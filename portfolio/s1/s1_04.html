Création d'une base de données

Description

Une agence de voyage « Little Booking » désire mettre en place un site web pour permettre à ses clients de réserver des hôtels, appart-hôtels et autres logements, réserver leurs vols ou encore réserver des séjours entiers (vols et hébergements compris). Pour ce faire, elle veut d’abord se prémunir d’un système d’informations correspondant à une base de données MySql permettant de décrire et manipuler toutes les structures de données dont elle a besoin pour la mise en place de son site.

Présentation

  • Nom du projet : Base De Données Little Booking
  • Mots-clés : SQL, MCD, MLD, Requêtes, Tables, Dictionnaire de données
  • Contexte : Entreprise informatique qui va concevoir une solution informatique permettant à l’agence de voyage de répondre au cahier des charges
  • But :
    • Implémentation d’une base de données qui devra prendre en compte la prospective pour le développement de l’agence et la mise à jour des stratégies marketing
    • Ci-dessous la liste non exhaustive des données économiques à renseigner dans la base de données
    • La solution proposée devra permettre de gérer les différentes structures de données de l’agence (ajout, suppression, modification). Par ailleurs, elle devra en autres permettre aux clients de rechercher des offres (séjours, hébergements, vols) selon les critères qui leur conviennent, aux hébergeurs de créer leurs hébergements, etc. Ceci afin de préparer l’étape suivante, celle de la mise en place du site web

Conditions de réalisation

  • Nombre d'étudiants : 3
  • Temps passé : 24h
  • Ressources utilisées : MySQL Workbench, Wampserver, Looping

Résultat final

  • Production finale du projet :
    • Le dictionnaire de données
    • Le MCD et modèle relationnel du projet
    • Les scripts SQL et les jeux d’essais (l’ensemble des requêtes pour manipuler la base de données). Les scripts comprennent des requêtes de complexité variable (simples, moyennes, complexes avec l’utilisation des jointures et des selects imbriqués)
    • Un rapport argumenté justifiant les choix techniques ayant été fait incluant les différents diagrammes générés. Les scripts peuvent et les jeux d’essais peuvent être mis en annexes
  • Qualités / Défauts / Améliorations : Cette base de données permet effectivement une organisation des différents éléments et liaisons composant l'agence de voyage; néanmoins, le travail réalisé aurait pu être amélioré avec notamment une amélioration des liaisons et des contraintes, une clarification des clés. Un test effectué avec davantage de données aurait également permis de consolider et diversifier les requêtes SQL incluses.
  • Évolutions possibles du projet : En terme d'évolution, davantage d'éléments pourraient être pris en compte pour enrichir la base de données; de plus, une interface utilisateur plus simple et intuitive serait une grande avancée pour le projet.

Apports

Ce projet m'a permis de développer mes compétences en langage SQL et en matière de création de base de données. Egalement, mes compétences en matière de logistique et d'organisation ont été sollicitées dans l'objectif de répondre au cahier des charges et composer une base de données fiable et solide. La méthodologie de création de base de données est nouvelle et requiert de partir d'une analyse détaillée et globale pour pouvoir réorganiser les données en une base exploitable. Les requêtes SQL représentent également une manière ciblée de collecter les informations assez intéressantes; leur complexité diverse est proportionnelle à leur utilisation.
