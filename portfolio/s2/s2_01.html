Développement d'une application

Description

Le but de ce projet est de développer un jeu vidéo en 2D : nous avons choisi de développer un jeu basé sur le Dino Runner de Google. Cette partie concerne le développement de l'application et a été majoritairement réalisée par un des deux membres de l'équipe (code : mon collègue; assets : moi-même).

Présentation

  • Nom du projet : T-Runner
  • Mots-clés : Mouvement, Ennemis, Score, Assets, Hitbox
  • Contexte : Projet du second semestre consistant en la réalisation d'un jeu-vidéo
  • But : La problématique professionnelle est la création de tout ou partie d’une application simple avec interface graphique. Cette SAÉ permet la concrétisation du développement autour d’une application avec une interface graphique répondant à un contexte précis

Conditions de réalisation

  • Nombre d'étudiants : 2
  • Temps passé : 50h
  • Ressources utilisées : Visual Studio Code ; LibreSprite ; Framework .NET Monogame pour C#

Résultat final

  • Production finale du projet : Un jeu vidéo 2D fonctionnel inspiré du Dino Runner de Google. Le joueur contrôle un personnage capable de sauter afin d'éviter différents obstacles (cactus et ptérodactyles). Le projet intègre les principales fonctionnalités attendues : gestion des déplacements, collisions, score, génération d'obstacles, écran de fin de partie et éléments graphiques réalisés avec LibreSprite.
  • Qualités / Défauts / Améliorations : Le jeu produit est fonctionnel et jouable, il utilise une interface simple et intuitive, et le framework Monogame a permis de simplifier la gestion d'éléments plus complexes sans pour autant fournir un moteur de jeu complet. Néanmoins, des défauts sont à noter, comme un problème de réglage entre la taille des cactus et la vitesse du joueur, pouvant potentiellement être réglé en améliorant la précision des hitbox.
  • Évolutions possibles du projet : La jouabilité du jeu est actuellement variable en fonction du bug cité précédemment ; il nécessiterait une correction pour régler le bug et appliquer d'autres potentiels patchs de corrections ou d'amélioration. Une autre évolution serait l'ajout de fonctionnalités de personnalisation de difficulté ou esthétique (initialement prévues à la base).

Apports

Concernant la partie Développement d'une application, ce projet nous a permis de développer nos compétences en programmation C# à l'aide du framework Monogame et notre capacité à résoudre des problèmes techniques. Nous avons appris à concevoir une application graphique, à gérer les déplacements et les collisions, à intégrer des ressources graphiques et à structurer un projet de développement.
